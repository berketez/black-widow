# Karadul Ghidra — Quick Reference

## Architecture at a Glance

### Two Analysis Modes
```
┌─ PyGhidra (Native Python 3 in JVM) ✅ PREFERRED
│  └─ Direct Ghidra API calls, better performance
│
└─ CLI (analyzeHeadless + Jython scripts) — Fallback
   └─ Subprocess-based, JVM per binary
```

### Analysis Output Pipeline
```
Binary Input
  ↓
[GhidraHeadless.analyze()]
  ├─ Functions (name, address, size, params) → functions.json
  ├─ Strings (value, xref count) → strings.json
  ├─ Call Graph (nodes, edges) → call_graph.json
  ├─ Decompilation (C code, disasm, stack) → decompiled.json + decompiled/*.c
  ├─ Type Info (structs, enums, typedefs) → types.json
  ├─ Xrefs (function/string/global refs) → xrefs.json
  └─ Combined Summary → combined_results.json
```

## Key Classes & Methods

### GhidraHeadless (headless.py)
```python
GhidraHeadless(config)
  .analyze(binary_path, project_dir, timeout, output_dir)
    → dict with scripts_output, duration_seconds, returncode
```

**Internal Methods**:
- `_analyze_pyghidra()` — Direct API
- `_analyze_cli()` — Fallback
- `_extract_*()` — Static extraction methods
- `_decompile_functions()` — Batch decompilation handler
- `_load_gdt_archives_pyghidra()` — Type enrichment

### GhidraProject (project.py)
```python
GhidraProject(workspace, config)
  .create() → project_dir
  .get_output_dir() → output_dir
  .get_decompiled_dir() → decompiled/
  .cleanup() → removes temp files
```

### StaticAnalysisStage (stages.py)
```
Stage → execute(context: PipelineContext)
  → analyzer.analyze_static(target, workspace)
    → GhidraHeadless.analyze(...)
      → Returns: StageResult with artifacts dict
```

## Common Operations

### Run analysis manually
```python
from karadul.ghidra import GhidraHeadless
from karadul.config import Config

config = Config.load()
headless = GhidraHeadless(config)
result = headless.analyze(
    binary_path="/path/to/binary",
    project_dir="/tmp/ghidra_project",
    timeout=7200,
    output_dir="/tmp/ghidra_output"
)
print(f"Success: {result['success']}")
print(f"Functions: {result['scripts_output']['functions']['total']}")
```

### Load and inspect results
```python
import json
from pathlib import Path

output_dir = Path("/tmp/ghidra_output")
functions = json.loads((output_dir / "functions.json").read_text())
strings = json.loads((output_dir / "strings.json").read_text())

print(f"Found {functions['total']} functions")
print(f"Found {strings['total']} strings")
print(f"Categories: {strings['category_stats']}")
```

### Query specific function
```python
call_graph = json.loads((output_dir / "call_graph.json").read_text())

# Find "main" function
main_node = next(
    (n for n in call_graph['nodes'].values() if n['name'] == 'main'),
    None
)
if main_node:
    print(f"main: {len(main_node['callees'])} callees")
```

## Troubleshooting

### Ghidra timeout on large binary
→ Increase `config.timeouts.ghidra` or set `large_binary_timeout_multiplier = 6.0`

### Out of memory during decompilation
→ Reduce `ghidra_batch_size` (e.g., 2500 or 1000)
→ Increase `ghidra_max_heap_mb` in config

### Type recovery poor quality
→ Add GDT archives matching target platform:
  - macOS: `"mac_osx.gdt"`
  - Linux: `"generic_clib_64.gdt"`
  - Windows: `"windows_vs12_64.gdt"`

### String extraction missing some strings
→ Reduce `analysis.string_min_length` from 4 to 2 (but increases false positives)

### PyGhidra not available
→ Falls back to CLI automatically
→ Check Ghidra installation path in config
→ Ensure `pyghidra` pip package installed in system Python
