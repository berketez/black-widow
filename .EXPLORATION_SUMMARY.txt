JSON OUTPUT FORMATS (7 MAIN + 1 COMBINED)
================================================================================

1. functions.json
   - Functions: name, address, size, param_count, return_type, params[]
   - Schema verified with real examples

2. strings.json
   - Strings with xrefs: address, value, function, xref_count
   - Categories: URLs, paths, API keys, error messages, format strings
   
3. call_graph.json
   - Nodes: {address: {name, caller/callee counts, lists}}
   - Edges: {from, to, from_name, to_name}
   - Analysis: root functions, leaf functions
   
4. decompiled.json
   - Summary: success/failed/skipped counts, duration
   - Per-function: name, address, file, lines, size, xrefs, stack_frame
   - Per-file: decompiled/{func_name}.c with header + code + disasm
   
5. types.json
   - Structures: name, fields[] {name, type, offset, size}
   - Enums: name, values[] {name, value}
   - TypeDefs: name, base_type
   - Category summary
   
6. xrefs.json
   - Function xrefs: strings_used, globals_accessed, functions_called, called_by
   - String xrefs: address, value, referenced_by[]
   - Global xrefs: readers/writers classification
   - Statistics: most referenced, most called, isolated functions
   
7. combined_results.json
   - Dashboard-friendly summary
   - Program info (language, processor, memory blocks)
   - All statistics aggregated
   - Samples (first 100 of each) with references to full files
